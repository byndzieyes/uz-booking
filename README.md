# Railway Booking (Система бронювання залізничних квитків)

Навчальний веб-застосунок для моделювання роботи системи продажу залізничних квитків (аналог Укрзалізниці)

## Функціонал

### Список потягів та рейсів

- Відображення списку доступних потягів у вигляді інтерактивних карток.
- Інформація про рейс: номер потяга, місто відправлення/прибуття, дата, час та тривалість поїздки.
- Пошук та фільтрація рейсів за номером потяга або назвами міст.

### Бронювання місць у вагоні

- Перехід на детальну сторінку конкретного рейсу.
- Вибір вагона з доступного списку.
- Інтерактивна схема місць у вагоні з візуальною індикацією статусів:
  - ⬜ Вільні
  - 🟦 Обрані користувачем
  - 🟥 Заброньовані
- Форма оформлення квитків із валідацією персональних даних (Ім'я, Прізвище, Телефон, Email).
- Збереження стану бронювання (persistence) за допомогою localStorage, що імітує роботу з базою даних.

## Технологічний стек

- **Frontend Framework:** React 19
- **Мова програмування:** TypeScript
- **Збирач:** Vite v8
- **Стилізація:** Tailwind CSS v4
- **Маршрутизація:** React Router v7
- **Збереження даних:** Web Storage API (localStorage)

## Встановлення та запуск

1. Склонуйте репозиторій на свій комп'ютер:
   git clone https://github.com/byndzieyes/uz-booking.git

2. Перейдіть у папку проєкту:
   cd uz-booking

3. Встановіть необхідні залежності:
   npm install

4. Запустіть проєкт у режимі розробки:
   npm run dev

5. Відкрийте локальне посилання (зазвичай http://localhost:5173) у вашому браузері.

## 📂 Структура проєкту

- `/src/components` — перевикористовувані UI компоненти (TrainCard, TrainList, SeatMap, BookingForm тощо).
- `/src/pages` — головні сторінки застосунку (Home, Booking).
- `/src/services` — логіка взаємодії з локальним сховищем (BookingService).
- `/src/data` — статичні "фейкові" дані для імітації API (trains.ts).
